The dashboard is the central hub you reach after authenticating. It is wrapped in the `ProtectedRoute` component, which verifies that a valid JWT exists in `localStorage` before rendering any content.

## Route protection

<Note>
  When you navigate to `/dashboard`, `ProtectedRoute` runs `getToken()` on mount. While the check is in progress it renders a **"Loading..."** message. If a token is found you see the dashboard. If no token is found you are immediately redirected to `/login`. This check is client-side only — it does not verify the token's signature or expiry.
</Note>

## Dashboard cards

The dashboard displays four navigation cards in a responsive grid:

  </Card>

<Warning>
  The pages linked from these cards — **Profile** (`/profile`), **Settings** (`/settings`), **Messages** (`/messages`), and **Analytics** (`/analytics`) — are not yet fully implemented. The links exist in the dashboard but the destination pages have no content.
</Warning>

Each card is rendered by the internal `DashboardCard` component, which wraps a Next.js `Link` and applies hover effects:

```jsx Dashboard.js
function DashboardCard({ title, children, link }) {
  return (
    <Link href={link} className="block">
      <div className="bg-white p-6 rounded-lg shadow-md hover:shadow-xl transition-shadow duration-300 ease-in-out transform hover:scale-105">
        <h2 className="text-xl font-semibold text-orange-500 mb-2">{title}</h2>
        <p className="text-gray-700">{children}</p>
      </div>
    </Link>
  );
}
```

Hovering over a card enlarges it slightly (`scale-105`) and deepens its shadow (`shadow-xl`) to give a lift effect.

## Logout

The red **Logout** button at the top of the dashboard removes your token from `localStorage` and sends you back to `/login`:

```js Dashboard.js
const handleLogout = () => {
  localStorage.removeItem('token');
  router.push('/login');
};
```

After logout, any attempt to reach a protected page will trigger an immediate redirect to `/login` via `ProtectedRoute`.
